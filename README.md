## 📌 한 줄 요약

이러닝 한 과정을 만드는 데 필요한 **네 가지 전문성**(컨설팅·교수설계·콘텐츠·운영)이 각자 다른 도구에서 분절되어 일하는 문제를, **표준 산출물 양식**을 공유하는 협업 플랫폼으로 풀어보았습니다.

---

## 💡 기획 배경

### 문제 정의

이러닝 한 과정이 만들어지기까지 최소 네 명의 전문가가 협업합니다.

| 역할 | 페인포인트 |
|---|---|
| 🎯 컨설턴트 | 제안서를 매번 처음부터 작성. 사례 자산이 개인 PC에 흩어져 있음 |
| 📋 교수설계자 | 스토리보드 양식이 회사마다, 프로젝트마다 다름 |
| 🎬 콘텐츠 제작자 | 영상 기획 단계에서 의사결정 지연. 컷 단위 협의가 어려움 |
| ✅ 운영관리자 | 체크리스트가 엑셀에 박제됨. 진행률·담당자 추적 어려움 |

### 해결 가설

> **"각 역할의 표준 산출물을 플랫폼 안에서 만들고 공유할 수 있다면, 협업의 시작점이 통일된다."**

단순한 커뮤니티가 아니라, **네 역할이 각자의 표준 양식을 직접 작성할 수 있는 협업 워크스페이스**로 설계했습니다.

## 🔧 핵심 기능

### 1. 네 가지 전문가 양식

각 카테고리에 진입하면 일반 글 외에 **전용 양식 에디터**가 제공됩니다.

| 카테고리 | 전용 양식 | 주요 필드 |
|---|---|---|
| 컨설팅 | 📄 **5섹션 제안서** | 배경분석 → 솔루션 → 일정 → 예산 → 기대효과 |
| 교수설계 | 📋 **스토리보드** | 씬 단위 / 학습목표 / 내레이션 / 상호작용 / 화면 이미지 |
| 콘텐츠 | 🎬 **영상기획서** | 컷 단위 / 샷 유형 / 듀레이션 / 자막 / BGM / 편집노트 |
| 운영 | ✅ **운영 체크리스트** | 기획 → 런칭 → 운영 → 평가 4단계 / 담당자 / 마감일 |

![storyboard](./screenshots/02-storyboard.png)

### 2. 통합 검색 + 태그 시스템
- 제목·본문·작성자·카테고리·태그 전체 검색
- 키워드 하이라이트, 실시간 자동완성, 최근/인기 키워드

### 3. 마이페이지
- 내 글 · 댓글 · 좋아요 · 북마크 통합 관리
- 사용자별 활동 통계

### 4. 알림 센터
- 내 글에 좋아요·댓글 발생 시 실시간 알림
- 헤더 배지로 미확인 알림 표시

### 5. 별도 운영 구인구직
- 인사이트 커뮤니티와 완전 분리된 게시판
- 채용 / 구직 / 북마크 탭 구성

### 6. 진행률 시각화 (체크리스트)
- 원형 도넛 차트 + 단계별 가로 막대
- 전체 진행률 % 자동 계산

## 🛠 기술 스택

**프론트엔드**: HTML5 + CSS3 + Vanilla JavaScript (단일 파일)
**저장소**: localStorage (인증, 게시글, 알림, 북마크)
**이미지 처리**: Canvas API (리사이즈 + JPEG 압축)
**PWA**: Service Worker (인라인 등록), Web Manifest
**외부 의존성**: 없음 (Google Fonts만 CDN)

> 본 프로젝트는 **기획자 포트폴리오** 목적으로, 모든 데이터는 브라우저 localStorage에 저장됩니다.

## 💬 기획자로서 배운 것

1. **양식은 협업의 시작점이다** — 도구가 통일되면 의사결정 속도가 다르다
2. **페르소나는 기능을 정당화한다** — "왜 이 기능이 필요한가"는 페르소나에서 출발
3. **분리할 것은 분리하자** — 커뮤니티와 구인구직처럼, 목적이 다르면 동선도 달라야 한다
4. **데이터는 자동저장되어야 한다** — 사용자는 실수하고, 잃어버린다
